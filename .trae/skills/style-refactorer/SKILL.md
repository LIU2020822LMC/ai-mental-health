# 样式重构器

这个 skill **只用于样式重构与样式制作**。

当用户有以下需求时，使用这个 skill：

- 重构页面、组件或全局样式
- 统一不一致的界面视觉风格
- 搭建或刷新一套视觉主题
- 优化布局、间距、排版、颜色、阴影、边框或动效
- 在不改变功能行为的前提下，让界面更简洁、更现代

以下情况 **不要** 使用这个 skill：

- 修改业务逻辑
- 修改数据流
- 修改状态管理
- 修改 API、路由、权限、校验或存储逻辑
- 重构组件行为

## 核心规则

1. 绝不修改任何业务逻辑代码。
2. 绝不改动事件处理、数据处理、请求、权限、路由或组件行为。
3. 如果某个视觉需求看起来必须依赖逻辑改动，先停止并询问用户。
4. 优先修改样式文件、主题 token、设计变量、仅用于表现层的类名，以及不影响功能的结构包装层。
5. 必须保留现有功能与交互语义。

## 固定设计方向

除非用户明确提出覆盖要求，否则统一采用以下风格方向：

- 风格：欧式、极简、现代化、清爽、海洋蓝
- 视觉气质：克制、优雅、通透、干净、理性
- 配色倾向：以海洋蓝、雾霾蓝、深海蓝、浅灰蓝为核心，辅以白色与低饱和中性色
- 布局倾向：留白充足、秩序清晰、比例平衡、减少视觉负担
- 组件倾向：轻边框、浅阴影、柔和圆角、清透表层、精致细节
- 动效倾向：轻柔、短促、自然，强调流畅与清爽，不做夸张演出

## 样式规范

### 色彩

- 使用以海洋蓝、深海蓝、雾蓝、灰蓝为核心的海洋系配色。
- 点缀色要克制且有明确目的，强调“清爽、通透、安静”的观感，而不是高饱和冲击感。
- 除非用户明确要求，否则避免高饱和彩虹式配色。
- 优先保证可读性和可访问性对比度。

示例配色方向：

- 背景：`#F4F8FC`、`#EDF4FA`、`#E7F0F8`
- 表层：`#FFFFFF`、`#F8FBFE`
- 主强调色：`#2F6FA3`
- 次强调色：`#5B8DB8`
- 深层辅助色：`#1F4F78`
- 主文字：`#17324A`
- 次文字：`#5F7890`
- 边框：`rgba(47, 111, 163, 0.14)`

### 排版

- 优先使用干净、现代的无衬线字体体系。
- 标题层级要清楚，表达要简洁。
- 避免装饰性过强的字体。
- 更偏向通过中等字重、舒展字距和稳定层级来体现欧式优雅感。

### 间距与布局

- 使用一致的间距尺度。
- 优先通过留白提升品质感，而不是增加额外装饰。
- 采用对齐清晰的栅格、均衡的内边距和明确的分组关系。
- 减少视觉噪音和冗余容器。

### 组件

- 卡片、面板、输入框保持简洁、克制、细致。
- 使用柔和圆角、浅层阴影和清透背景层次。
- 用海洋蓝边框、高光或浅色底强调聚焦态与激活态。
- 按钮要清晰、易读、视觉上平静稳定，并避免厚重压迫感。

### 动效

- 悬停、聚焦、面板切换使用短时过渡。
- 避免夸张动画。
- 动效的目标是辅助理解，而不是主导界面。

## 工作流程

使用这个 skill 时，请遵循以下流程：

1. 先识别哪些文件只影响表现层。
2. 把纯样式改动和任何逻辑相关代码彻底分离。
3. 只重构样式层。
4. 在本次涉及的 UI 范围内统一颜色、间距、圆角、阴影和排版。
5. 检查并确认没有改动任何业务逻辑。
6. 总结结果时，用设计与视觉语言描述改动，而不是用逻辑语言描述。

## 输出要求

在完成基于这个 skill 的工作后，必须明确确认：

- 只修改了样式
- 没有改动业务逻辑
- 最终界面符合欧式、极简、现代化、清爽、海洋蓝的统一方向

## 触发示例

当用户说出类似下面的话时，应调用这个 skill：

- “帮我重构一下这个页面样式”
- “把整个 UI 风格统一一下”
- “只重构样式，不要动逻辑”
- “做得更极简、更现代一点”
- “把这个项目改成海洋蓝设计语言”
- “整体做成欧式一点、清爽一点”
- “优化视觉设计，但不允许修改业务代码”
